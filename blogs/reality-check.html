Article 3: Real-World Problem Solving

The gap between a "project that works" and a "project that is deployed" is where the real learning happens. Here are the hurdles I faced when taking Emergency QR live.

Challenge 1: The CORS & Cookie Nightmare

Once deployed, cookies containing my JWT were being dropped because of cross-origin policies.

The Fix: Explicitly white-listed the Netlify domain, set withCredentials: true in Axios, and updated cookie headers to SameSite: 'none' and Secure: true.

// The Production Fix
res.cookie('token', token, {
  httpOnly: true,
  secure: true,    
  sameSite: 'none', 
  maxAge: 7 * 24 * 60 * 60 * 1000 
});

Challenge 2: The "Broken" Deployment Build

Deployment failed on Render because the client-side build wasn't triggered correctly in the monorepo.

The Fix: Implemented a Post-Install Script to build React production files before the Node server start.

Challenge 3: Environment Variable Mismatch

Vite handles environment variables differently. My frontend couldn't see the backend URL because of prefixing rules.

The Fix: Prefixed variables with VITE_ and synchronized secrets manually in the platform dashboards.

My Debugging Framework

  1. Isolate: Determine the scope of the bug.
  2. Reproduce: Confirm the behavior consistently.
  3. Research: Consult documentation (MDN, Vite).
  4. Verify: Test the fix in staging.
